At some point tutorials cap out. The next unlock is reading production code written by people better than you. Here's how I do it.

## Pick something you already use

I read `shadcn/ui`, `zod`, and pieces of Next.js. If you use it every day, you already have half the context.

## Start from the entry point, not the README

Open `package.json`, find `main` or `exports`, follow the imports. This teaches you the shape of the library — which parts are public API and which are internal.

## Read tests before implementation

Tests tell you what the code is *supposed* to do. The implementation tells you how. Start with intent.

## Trace one feature end to end

Pick a feature (e.g. "how does `useForm` handle async validation"), find where the user calls it, and follow the code all the way down until nothing surprises you.

## Take notes in your own words

Not copy-paste. Actually rephrase what a function does. If you can't rephrase it, you don't understand it yet.

An hour of this a week beats a month of passive tutorials. Try it.
