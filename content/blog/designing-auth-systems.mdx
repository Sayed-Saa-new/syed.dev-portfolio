Auth is one of those things everyone thinks is easy until they've been paged at 3 AM because sessions won't refresh. Here are the rules I now follow on every project.

## 1. Never roll your own primitives

Use Argon2id (or bcrypt at minimum) for passwords. Use the platform's session library. If you're writing your own JWT signing loop, stop.

## 2. Short access tokens, long refresh tokens, rotated

15-minute access tokens, refresh tokens rotated on every use, replay detection on the server. This is the standard for a reason.

## 3. Store roles in a separate table

Never put `is_admin` on the user row. It's the fastest path to a privilege escalation bug. Roles belong in `user_roles` with a security-definer function checking them.

## 4. Rate limit before you need to

IP + user + endpoint. Add it on day one, not after the incident.

## 5. Log auth events, but not secrets

Every login, logout, password reset, MFA challenge. Never the token. Never the password. If a log line would be scary in a breach dump, don't log it.

Small list. Non-negotiable. Skip any of them and you'll pay for it eventually.
