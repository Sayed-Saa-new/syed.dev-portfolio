Not a beginner list. These are the patterns that made my TypeScript feel calm instead of angry.

## `satisfies` instead of type annotations

```ts
const config = {
  mode: "dark",
  theme: "aegis",
} satisfies AppConfig;
```

You get the constraint check *and* the narrow inferred type. Annotations widen. `satisfies` doesn't.

## Discriminated unions for state

```ts
type Fetch<T> =
  | { status: "idle" }
  | { status: "loading" }
  | { status: "success"; data: T }
  | { status: "error"; error: Error };
```

TypeScript narrows perfectly on `switch (state.status)`. No more optional chaining hell.

## Template literal types for string enums

```ts
type Route = `/blog/${string}` | `/projects/${string}`;
```

Great for typed router paths and event names.

## `Awaited<ReturnType<typeof fn>>` for async return types

Cleaner than manually threading `Promise<...>` through helpers.

## `NoInfer` when generics guess wrong

Rarely needed, but a lifesaver when you're writing a library and want the caller to pass a hint without the compiler over-inferring from it.

Small tools. Big compounding return.
